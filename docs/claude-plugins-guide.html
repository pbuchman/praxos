MCP Servers Model Context Protocol

context7

docs

Fetch up-to-date documentation and code examples for any programming library or framework directly from their official sources.

Available Tools

resolve-library-id Convert a library name to a Context7-compatible ID
query-docs Retrieve documentation for a specific library
Pro Tip

Always call resolve-library-id first to get the correct ID before querying docs. Limit to 3 calls per question.

Example Usage 
// Step 1: Resolve the library ID
resolve-library-id({
  "libraryName": "fastify",
  "query": "How to set up routes with validation"
})

// Step 2: Query the documentation
query-docs({
  "libraryId": "/fastify/fastify",
  "query": "Route validation with JSON schema"
})

playwright

browser

Browser automation for testing, screenshots, and web interactions. Control Chrome, Firefox, or WebKit programmatically.

Key Tools

browser_navigate Navigate to a URL
browser_snapshot Capture accessibility snapshot (preferred over screenshot)
browser_click Click an element by reference
browser_type Type text into an input field
browser_take_screenshot Capture visual screenshot
browser_fill_form Fill multiple form fields at once
Important

Use browser_snapshot for actions (returns element refs). Use browser_take_screenshot only for visual verification.

Web Automation Flow 
// 1. Navigate to page
browser_navigate({ "url": "https://example.com/login" })

// 2. Get page snapshot with element refs
browser_snapshot({})

// 3. Fill the login form
browser_fill_form({
  "fields": [
    { "name": "email", "type": "textbox", "ref": "input[0]", "value": "user@example.com" },
    { "name": "password", "type": "textbox", "ref": "input[1]", "value": "secret" }
  ]
})

// 4. Click submit button
browser_click({ "element": "Submit button", "ref": "button[0]" })

claude-mem

memory

Persistent memory system that records observations, decisions, and learnings across sessions. Essential for maintaining context in long-running projects.

3-Layer Workflow (CRITICAL)

Always follow this pattern to save tokens: search → timeline → get_observations. Never fetch full details without filtering first.

1
search(query)

Get index with IDs (~50-100 tokens/result). Fast initial filtering.

2
timeline(anchor=ID)

Get context around interesting results. See temporal relationships.

3
get_observations([IDs])

Fetch full details ONLY for filtered IDs. This is where the tokens go.

Search Parameters

query Semantic search term
type bugfix, feature, refactor, discovery, decision, change
project Filter by project name
dateStart/dateEnd Date range filtering
Memory Search Pattern 
// Step 1: Search for relevant memories
search({
  "query": "cost tracking LLM",
  "type": "bugfix",
  "limit": 10
})
// Returns: [{id: 1585, title: "..."}, {id: 1586, title: "..."}]

// Step 2: Get context around a specific result
timeline({
  "anchor": 1585,
  "depth_before": 3,
  "depth_after": 3
})

// Step 3: Fetch full details for relevant IDs only
get_observations({
  "ids": [1585, 1586, 1587]
})

Systematically improve test coverage to meet the 95% threshold. Uses a tiered approach with continuity tracking.

1
Check Continuity

Look for existing CONTINUITY.md ledger to resume work

2
Investigate

Run coverage analysis and identify gaps

3
Create Structure

Generate tiered subtask files (Tier 1-4 by effort)

4
Work Continuously

Implement tests tier by tier until threshold met

Non-Negotiable

NEVER modify vitest.config.ts thresholds. Write tests instead. Coverage target is 95%.

quality

Detect and fix code smells. Enforces architecture hygiene and updates CLAUDE.md with new patterns.

Priority Examples

P1-P3 Architecture violations, DI issues

P7-P9 Style issues, minor improvements
One Fix Per Pass

The command fixes one smell at a time, runs npm run ci, and updates CLAUDE.md if a new pattern is discovered.

💡 Today's Insight: Firestore Composite Indexes

Did you know? Multi-field queries in Firestore require
composite indexes, or they'll fail silently in production.

In IntexuraOS, we define indexes in migrations:

export const indexes = [{
  collectionGroup: 'bookmarks',
  fields: [
    { fieldPath: 'userId', order: 'ASCENDING' },
    { fieldPath: 'createdAt', order: 'DESCENDING' }

}];
